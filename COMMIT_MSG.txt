SUMMARY
══════════════════════════════════════════════════════════════════

Complete local web application for exploring and analysing C codebases
through an interactive SVG call graph. Replaces the cflow+gawk+ctags
pipeline with a pure-Python tree-sitter analyzer that extracts function
calls, global variable dependencies, ISR/task classification, RTOS
architecture (FreeRTOS + CMSIS-RTOS v2), and hardware peripheral
register access from real embedded firmware. No cloud. No telemetry.
No system dependencies beyond Python, Flask, and tree-sitter.

Tested on two real projects — EVIC (69 files, 552 functions, 0.3 s)
and HMI (206 files, 570 functions, 2.4 s). 64 / 64 automated tests
passing across 7 test sections covering parser, analyzer, graph schema,
cross-project consistency, and JS frontend logic.

ANALYZER: TREE-SITTER 7-PASS ENGINE  (replaces cflow+gawk)
══════════════════════════════════════════════════════════════════

  Old pipeline: cflow + gawk + ctags
    - misses function pointers, struct dispatch, ifdef branches
    - fragile text parsing
    - system binary dependencies (cflow must be installed)
    - no global variable tracking
    - no ISR/race analysis

  New pipeline: tree-sitter + tree-sitter-c (pip only, no system deps)
    - AST-based, handles complex embedded C
    - correct scope resolution (file-scope vs local)
    - field access detection (GPIOA->ODR)
    - identifier context: LHS vs RHS, inside loop body, etc.

  Pass 1  Global collection
          Scans all .c files for file-scope declarations.
          Extracts: name · file · line · volatile · static · extern

  Pass 2  Per-function body analysis
          Walks AST of each function_definition node.
          Collects: call_expression nodes → callees
                    identifier nodes matching globals → reads
                    assignment LHS / compound-assign / update_expression → writes
                    field_expression base names → peripheral register access
                    known RTOS API calls → kind + target argument
                    critical section entry calls → has_critical flag
                    delay calls inside for/while/do → delay_in_loop flag

  Pass 3  Call graph construction
          edges dict: caller → [callee, ...]
          callers dict: callee → [caller, ...]
          out_degree, in_degree per node

  Pass 4  Global variable dependency map
          For each global: { writers: [fn...], readers: [fn...], rw: [fn...] }

  Pass 5  Race condition detection
          Crosses ISR-named function write-sets with non-ISR access-sets
          for each global variable. Checks for critical section entry.
          Severity: HIGH (volatile, unprotected)
                    MEDIUM (non-volatile, unprotected)
                    LOW (has critical section)

  Pass 6  RTOS architecture graph
          44 FreeRTOS + CMSIS-RTOS v2 API patterns.
          Extracts task function names, queue handles, mutex/semaphore
          handles from call arguments. Builds per-task relationship map
          and shared-object inventory.
          Filters FreeRTOS internals: prv* prefix, NULL, struct field
          access targets (thread_def->pthread).

  Pass 7  Peripheral register map
          Matches field_expression base names against PERIPHERAL_RE.
          Distinguishes write (LHS of assignment), read (RHS), and
          read-modify-write (compound assignment).

ANALYSIS TOOLS
══════════════════════════════════════════════════════════════════

  ☠ Dead code
    Algorithm: zero project-in-degree nodes
    Excludes:  main · ISR_* · irq_* · *_irq · *_isr · interrupt_*
               handler_* · init_* · _init · startup · reset_handler
               _start · setup
    Output:    graph overlay (red + dim others) + .txt export

  ⚡ Change impact
    Input:     paste changed function names (one per line or CSV)
    Algorithm: reverse BFS upward through caller graph
    Output:    total affected count · per-module breakdown
               graph overlay (red = changed, orange = affected)
    UI state:  ◁ Edit returns to input without losing results

  🌡 Complexity heatmap
    Score:     (out_degree × 2) + in_degree + call_depth
    call_depth: BFS distance from nearest root function
    Thresholds: green < 33% < yellow < 66% < red (of project max)
    Output:    top-10 list + optional graph overlay

  ⇢ Path finder
    Algorithm: DFS with visited-set (cycle prevention)
    Limits:    max depth 12 hops · max 20 paths returned
    Sort:      shortest first
    Output:    clickable path list → step-numbered graph render

  ⊕ Intersection trace
    Algorithm: full descendant BFS from both functions → set intersect
    Output:    subgraph of shared callees only

EMBEDDED ANALYSIS PANELS  (new in this release)
══════════════════════════════════════════════════════════════════

  Globals panel  (graph bar — green)
    - All file-scope variables: name · file · line · volatile flag
    - Per-variable: write count · read count · RW count
    - Per-function breakdown expandable per variable
    - Click variable → highlights: writers red · readers green
      RW purple · others dimmed in graph
    - Buttons hidden when no globals in project

  Races panel  (graph bar — red, shown only when races detected)
    - Severity ranking: HIGH > MEDIUM > LOW
    - HIGH:   volatile global, ISR writes, non-ISR accesses,
              no critical section detected in non-ISR function
    - MEDIUM: non-volatile global, same unprotected pattern
    - LOW:    same pattern but critical section detected
    - Click candidate → ISR node red · task node orange in graph
    - ISR classification: 30+ name patterns covering prefixes,
      STM32 peripheral patterns, ARM fault handlers, and suffixes

  RTOS panel  (graph bar — purple, shown only when tasks detected)
    - 44 FreeRTOS + CMSIS-RTOS v2 API names parsed
    - Per task creator: creates · sends_to · recvs_from
                        takes · gives · delays
    - Shared objects: queues · mutexes · semaphores with user lists
    - FreeRTOS internals filtered: prv* prefix, prvIdleTask,
      thread_def->pthread struct access

  Periph panel  (graph bar — gold, shown only when peripherals found)
    - Detected via PERIPH->REGISTER field_expression patterns
    - GPIO A–K · TIM 1–14 · USART/UART 1–6 · SPI 1–6 · I2C 1–3
      DMA 1–2 · ADC · DAC · RCC · FLASH · PWR · SYSCFG · EXTI
      NVIC · SCB · SysTick · CAN · USB · RTC · IWDG · WWDG
      LTDC · SAI · QUADSPI · SDIO
    - Write / Read / RW per function per peripheral
    - Click peripheral → highlights all touching functions in graph

  Topbar stats (displayed after index):
    "{N} fns · {M} edges"
    + "{R} races" (red)  if races > 0
    + "{T} tasks" (purple)  if RTOS tasks > 0

BUG FIXES
══════════════════════════════════════════════════════════════════

  #1  clearTrace() did not reset multiTraceMode / multiTraceA / B
      → state leaked, stale intersection view after mode switch

  #2  renderGraph() re-rendered full graph instead of staying in
      intersection mode
      → added early-return guard that delegates to runMultiTrace()

  #3  runMultiTrace() called clearTrace() internally
      → clearTrace() triggered renderGraph() → runMultiTrace() loop
      → replaced with inline state reset (tracedFn=null, history=[])

  #4  newIndex() did not reset multiTraceMode, multiTraceA/B,
      traceHistory, window._deadFns, window._impactFns,
      window._heatScores, window._foundPaths
      → session state leaked across project switches

  #5  newIndex() did not clear #analysis-result innerHTML
      → stale analysis panel visible after switching projects

  #6  showImpactInGraph() missing null guard for window._changedFns
      → TypeError crash when called before runImpact() completes

  #7–9  Analysis overlay timing: 300 ms timeout too short for async
      renderGraph (20 ms spinner + Sugiyama layout + SVG build)
      → showDeadInGraph, showImpactInGraph, applyHeatmap
      → increased to 600 ms + error toast if SVG not ready

  #10  ISR pattern too narrow
      irq_uart_rx, uart_irq, spi_isr not matching.
      Pattern only covered irq_ prefix, not _irq/_isr suffixes.
      → extended ISR_RE: added .*_irq$ and .*_isr$ suffix patterns

  #11  RTOS target parser stored struct field access expressions
      as task function names: thread_def->pthread
      → filter: skip any target containing -> or . characters

  #12  FreeRTOS internal functions stored as project task nodes
      prvIdleTask, prv* prefix functions
      → filter: skip targets matching prv* prefix or in internal set

  #13  JS runHeatmap() const depth declaration
      Conflicted with app-global `depth` variable (slider value)
      in flat concatenated test harness, causing SyntaxError.
      → renamed: hmDepth · hmQ · hmVis throughout runHeatmap()
