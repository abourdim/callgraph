── BUG FIXES ─────────────────────────────────────────

[FIX 1] Dead code false positives on ISRs and entry points
  - Previously only name-pattern matching (main, ISR_, init_, etc.)
  - Now respects analyzer-computed is_isr and is_entry node flags
  - Eliminates false positives for handlers like TIM2_IRQHandler

── ANALYSIS FEATURES ─────────────────────────────────

[ANA 1] Cycle Detection (Tarjan SCC)
  - New "↻ Cycles" analysis button
  - Tarjan's SCC algorithm on full project call graph
  - Reports mutual call cycles (≥2 nodes) and self-recursive functions
  - Clickable cycle members → trace function
  - "Highlight in graph" button: rings cycle nodes red, fades others
  - Confirms DAG structure when no cycles found (prerequisite for stack depth)

[ANA 2] Worst-Case Stack Depth
  - New "⬆ Stack depth" analysis button
  - DFS from every detected entry point (ISR, main, init_*, is_entry, is_isr)
  - Cycle-breaking DFS — produces lower bound when cycles exist
  - Top 8 deepest chains shown, colour-coded: green <8, amber 8–12, red >12
  - "Show chain" button highlights the full chain in the graph (purple, graduated)
  - Depth counts critical for RTOS task stack sizing and MISRA compliance

── WORKFLOW IMPROVEMENTS ─────────────────────────────

[UX 1] JSON graph export
  - New JSON button in graph toolbar
  - Exports: all node metadata (ISR, critical, peripheral, globals, RTOS, file:line),
    all directed edges, full globals map, race candidates
  - Machine-readable; suitable for CI scripts, coverage tools, custom analysis

  - New CSV button in graph toolbar
